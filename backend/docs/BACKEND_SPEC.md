# 后端 MVP 产品说明

> Aftergift Phase 2 后端 MVP | 版本：1.0

---

## 1. 后端 MVP 目标

Phase 2 后端的核心目标：

1. **故事持久化**：用户发布的礼物故事存储到数据库，不再仅存在于前端内存
2. **匿名身份**：用户不需要提供真实姓名、手机号，使用系统生成的匿名昵称
3. **内容安全**：所有故事经过 AI 审核 + 人工复审，不直接公开高风险内容
4. **收藏同步**：登录用户可在任何设备上查看和管理自己的收藏
5. **举报处理**：用户可以举报疑似违规内容，管理员介入处理

---

## 2. 核心对象

### 2.1 用户（User）

- **匿名用户**：无需注册，可直接发布故事（生成随机匿名昵称）
- **可选登录**：用户可选择绑定手机号（用于跨设备收藏同步），手机号 HASH 存储
- **管理员**：后台审核人员，有额外的审核管理权限
- **禁止**：平台不提供用户间的真实身份查询

### 2.2 礼物（Gift）

- **主体**：代表一件关系旧物的核心对象
- **关联**：用户（发布者）、故事（story）、收藏记录（favorites）、举报（reports）
- **状态**：pending_review → published / needs_edit / rejected / archived

### 2.3 故事（GiftStory）

- **一句话故事**：100 字以内，显示在卡片摘要
- **完整故事**：600-2000 字，显示在 Modal 详情页
- **审核信息**：包含 quality_score、risk_level、suggestions

### 2.4 收藏（Favorite）

- **关联**：用户 + 礼物（多对多）
- **显示**：用户可查看自己收藏的所有礼物
- **限制**：同一用户对同一礼物只能收藏一次

### 2.5 举报（Report）

- **触发**：任何访客可举报疑似违规内容（无需登录）
- **处理**：进入管理员复审队列
- **结果**：下架故事、标记用户、忽略举报

### 2.6 审核记录（ReviewLog）

- **记录**：每次 AI 审核和人工审核的详细结果
- **保留**：审核日志不可删除，用于合规审计
- **可见性**：仅管理员可查看

---

## 3. 核心能力

### 3.1 发布礼物故事

```
用户提交 → 服务端接收 → 规则预检 → AI 审核 → 风险分级
  → [高风险] 进入人工复审队列
  → [中风险] 返回修改建议给用户
  → [低风险] 直接发布
```

**提交字段**：
- title（礼物名称）
- category（类型）
- relation_type（关系类型）
- action_type（处理方式）
- emotion（情绪标签）
- price_or_exchange（价格或交换意向）
- short_story（一句话故事）
- full_story（完整故事）
- is_anonymous（是否匿名，默认 true）

**响应**：
- 返回 gift_id 和审核状态
- 用户可通过 gift_id 查询审核结果

### 3.2 获取故事列表

- **公开列表**：所有 status=published 的礼物（匿名展示）
- **筛选**：action_type / emotion / favorites（我的收藏）
- **分页**：cursor-based pagination，避免大偏移量

### 3.3 获取故事详情

- **公开访问**：published 状态的故事任意可查看
- **自己发布的故事**：可查看 pending / needs_edit 等状态
- **安全提示**：详情页包含平台安全声明

### 3.4 收藏 / 取消收藏

- **需要登录**：收藏需要绑定用户身份（手机号 HASH）
- **跨设备同步**：同一手机号 HASH 的用户可在任何设备访问收藏
- **数量限制**：无上限

### 3.5 举报内容

- **无需登录**：任何人都可以提交举报（记录 IP hash）
- **举报原因**：用户选择或输入
- **处理流程**：举报 → 标记故事 → 进入复审 → 结果通知

### 3.6 审核状态流转

```
draft（用户草稿）
  ↓ POST /api/gifts
pending_review（审核中）
  ↓ AI 审核 + 人工复审
  ├── safe → published（公开）
  ├── caution → needs_edit（用户修改后重新提交）
  └── high_risk → rejected（拒绝，用户收到通知）
  ↓
archived（用户删除或管理员下架）
```

## 4. 内容发布原则

### 4.1 默认进入待审核

所有新发布的故事不会直接公开，必须经过：
1. 服务端规则预检（过滤明显违规）
2. AI 审核层（风险分级）
3. 高风险内容人工复审

### 4.2 高风险内容不直接公开

以下内容被识别为高风险时，直接拒绝发布，并通知用户修改：
- 直接暴露真实身份信息（姓名+电话+地址）
- 明显的报复、羞辱、诅咒内容
- 可识别特定关系对象的详细描述

### 4.3 用户身份匿名化

- 所有故事默认匿名展示（发布者只显示系统生成的随机昵称）
- 用户可选择公开自己的昵称（而非真实姓名）
- 管理员后台可看到手机号 HASH（用于多设备同步，不展示明文）

### 4.4 故事优先，交易其次

- 平台的价值观是「故事比交易更重要」
- 即使礼物状态为「出售中」，用户首先是为了讲一个故事
- 交易意向只是流转方式，不是平台的核心价值

### 4.5 不做真实交易闭环

Phase 2 后端：
- 记录用户的「想要这件」意向
- 提供交换匹配的基础逻辑
- **不接入支付、不接入物流、不做资金托管**

真实交易闭环在 Phase 4 单独评估。

---

## 5. 非目标（产品边界）

| 非目标 | 说明 |
|--------|------|
| 不是社交平台 | 禁止评论、点赞、关注等社交功能 |
| 不是真实交易平台 | 不处理支付、物流、资金托管 |
| 不提供真实身份查询 | 无法通过手机号、昵称找到真实用户 |
| 不存储明文敏感数据 | 手机号 HASH，不存真实号码 |
| 不允许上传图片（含他人照片）| Phase 3 之前图片功能不开放 |
