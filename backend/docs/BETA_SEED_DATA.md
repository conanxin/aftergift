# Aftergift Beta Seed Data 指南

## 目标

为小范围内测准备 20–30 条虚构示例礼物数据，覆盖多种关系类型和处理方式。

## 推荐规模

- **总数**：20–30 条
- **已发布**：15–20 条（供浏览和搜索）
- **审核中**：3–5 条（测试审核流程）
- **需修改**：2–3 条（测试 resubmit 流程）
- **已归档**：2–3 条（测试 restore 流程）

## 关系类型分布

| 关系 | 数量 | 示例 |
|------|------|------|
| 恋人/前任 | 8–10 | 戒指、项链、情侣装 |
| 朋友 | 5–6 | 书、手账本、耳机 |
| 家人 | 3–4 | 围巾、杯子、台灯 |
| 同事 | 2–3 | 钢笔、日历、植物 |
| 其他/未说明 | 2–3 | 相机、背包、音乐盒 |

## 处理方式分布

| 方式 | 数量 | 说明 |
|------|------|------|
| 出售 | 6–8 | 标价格 |
| 交换 | 4–5 | 写交换意向 |
| 赠送 | 4–5 | 免费送出 |
| 捐出 | 2–3 | 捐给公益机构 |
| 只展示故事 | 4–5 | 不交易，只讲故事 |

## 故事写作原则

1. **匿名**：不出现真实姓名、昵称、社交账号
2. **不曝光**：不出现具体地址、公司、学校、电话
3. **不攻击**：不侮辱、控诉、羞辱关系对象
4. **温柔克制**：聚焦物品和自己的感受，而非审判他人
5. **真实感**：用具体细节（气味、场景、时间）营造真实感，但保持虚构

## 示例故事片段

**出售 — 戒指**
> "在一起两年，分手时他把这个留在抽屉里。我戴了三个月才摘下来。现在它应该去一个不会提醒我的地方。"

**赠送 — 书**
> "她送我这本《挪威的森林》，说主角很像她。后来我们不再联系，书还像新的一样。希望它能在别人手里被读完。"

**只展示故事 — 台灯**
> "奶奶留下的。搬家三次都带着，但新家的灯位不适合它。不打算卖，只是想记下：它亮过很多个我写作业的夜晚。"

## Seed 插入方式

### 方式一：直接 SQL（推荐）

```bash
cd backend/backend
sqlite3 aftergift_dev.db < schema/seed_data.sql
```

### 方式二：通过 API

启动后端后，用 curl 或前端表单逐个创建。

### 方式三：批量脚本

可编写 Python 脚本读取 JSON 批量插入（如需，后续提供）。

## ⚠️ 重要边界

- **不要把真实用户故事提交到 Git**
- **seed_data.sql 中的故事必须是虚构的**
- **不要包含任何可识别真实个人的信息**
- **内测期间收集的真实故事也不要提交到代码仓库**
