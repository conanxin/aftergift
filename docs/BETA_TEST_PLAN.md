# Aftergift 小范围内测计划

## 内测目标

验证 Aftergift 本地 MVP 在真实用户场景下的可用性、理解度和情感安全性。

## 推荐人数

**5–10 人**

- 2–3 人：有分手/关系结束经历，愿意分享故事
- 2–3 人：对二手交易/旧物处理感兴趣
- 1–2 人：对产品/设计敏感，能提供结构反馈

## 测试路径

### 路径 A：首次体验（必做）

1. 打开 `http://127.0.0.1:8080/?api=local`
2. 浏览首页，理解产品定位
3. 创建匿名身份
4. 发布一件礼物（填写完整表单）
5. 查看「我的发布」
6. 尝试编辑故事
7. 尝试归档礼物
8. 尝试恢复归档礼物

### 路径 B：内容发现（必做）

1. 使用搜索框搜索关键词
2. 按「出售」「交换」「赠送」筛选
3. 点击礼物卡片查看详情
4. 收藏一个故事
5. 查看「我的收藏」

### 路径 C：管理员视角（可选）

1. 打开 `http://127.0.0.1:8080/?api=local&admin=1`
2. 输入 admin token 查看审核队列
3. 对一条待审核内容做出决定

## 观察指标

| 指标 | 观察方式 |
|------|----------|
| 是否理解这不是前任曝光平台 | 访谈第一感受 |
| 是否愿意写完整故事 | 观察填写完整度 |
| 是否理解匿名规则 | 检查是否尝试填写真实姓名 |
| 是否能找到自己的内容 | 观察「我的发布」使用频率 |
| 是否能处理审核退回 | 观察 needs_edit 后的行为 |
| 是否误以为是二手交易平台 | 检查是否只填价格不写故事 |

## 风险观察

| 风险信号 | 应对 |
|----------|------|
| 报复性表达 | 审核拒绝 + 私下沟通 |
| 身份暴露 | 审核打回修改 + 提醒匿名规则 |
| 误当二手平台 | 首页概念说明 + 表单提示 |
| 情感不适 | 提供「暂时收起」功能 |
| 内容质量低 | 审核层过滤 + 引导文案 |

## 内测后整理

1. 收集反馈表（见 BETA_FEEDBACK_FORM.md）
2. 整理常见问题 TOP 5
3. 记录最严重 bug
4. 决定是否进入 Phase 2I（内容推荐）或修复后重新内测
5. 更新 KNOWN_ISSUES.md
