# Aftergift 使用说明

**Phase closeout 文档 | 本文档供最终用户和产品演示参考**

---

## 1. 普通用户体验路径

### 1.1 首页浏览

打开前端（静态模式或 API 模式），首页展示礼物故事流。

默认展示全部礼物，支持按处理方式筛选：
- **全部**：展示所有已发布的礼物
- **出售**：可购买/带价的礼物
- **交换**：愿意用物品交换的礼物
- **赠送**：愿意免费送出的礼物
- **捐出**：愿意捐出的礼物
- **只展示故事**：暂不出售/交换/赠送，仅展示故事

### 1.2 搜索故事

使用顶部分类按钮或页面内搜索框：
- 按礼物名称关键词搜索
- 按关系类型筛选（恋人/夫妻/朋友/家人）
- 按情绪标签筛选（放下/遗憾/感谢/释怀/重启/纪念）

### 1.3 查看礼物详情

点击任意礼物卡片，打开详情 Modal：
- 礼物名称、类型、关系类型、处理方式、情绪标签
- 一句话故事摘录
- 完整故事正文
- 价格或交换意向（如有）
- 安全提示（匿名提醒）

### 1.4 收藏故事

在礼物详情 Modal 中点击 **收藏这个故事** 按钮。
收藏后，礼物会出现在"我的收藏"页面。

> 收藏功能需要在 API 模式下使用（`?api=local`）。

### 1.5 进入我的收藏

访问：

```
http://127.0.0.1:8080/?view=favorites
```

展示所有已收藏的礼物卡片，支持移除收藏。

### 1.6 创建匿名身份

在 API 模式下，前端会自动创建匿名 JWT 身份（存储在 localStorage）。

无需手机号、邮箱、社交账号。

### 1.7 发布礼物故事

1. 在首页点击 **发布一件礼物** 按钮
2. 填写表单（礼物名称、类型、关系类型、处理方式、情绪标签、价格/交换意向、一句话故事、完整故事）
3. 选择是否匿名
4. 点击提交

> 前端会进行敏感信息检测，如有风险会提示修改。
> 发布后礼物进入 `pending_review` 状态，需 Admin 审核后方可展示。

### 1.8 进入我的发布

访问：

```
http://127.0.0.1:8080/?view=me
```

展示当前用户发布的所有礼物，包括：
- 已发布（通过审核）
- 待审核
- 需要修改
- 已归档

支持操作：编辑并重新提交、归档、恢复。

### 1.9 管理草稿

发布表单填写过程中，内容会自动保存为草稿。

访问草稿管理：

```
http://127.0.0.1:8080/?view=drafts
```

展示所有草稿，包括：
- 草稿名称/标题
- 保存时间
- 超过 30 天的草稿会显示过期提示

支持操作：恢复编辑、删除（需二次确认）。

### 1.10 Admin 审核流程

以 Admin 身份访问：

```
http://127.0.0.1:8080/?api=local&admin=1
```

使用 Admin Token 登录后，可操作：
- **举报队列**：查看被举报的礼物，处理（approve/needs_edit/reject）
- **待审核队列**：审核新发布的故事
- **操作历史**：查看所有审核操作的日志

## 2. URL 参数参考

| URL | 说明 |
|-----|------|
| `/` | 首页，礼物故事流 |
| `/?view=favorites` | 我的收藏 |
| `/?view=me` | 我的发布（需 API 模式）|
| `/?view=drafts` | 我的草稿 |
| `/?api=local` | 启用 API 模式（连接本地后端）|
| `/?api=local&admin=1` | Admin 模式（审核面板）|

## 3. 注意事项

### 3.1 不要写什么

**禁止在故事中暴露**：
- 真实姓名（自己的和对方的）
- 电话号码、手机号
- 微信、QQ、邮箱、社交账号
- 家庭地址、公司地址
- 车牌、订单号、快递单号
- 能够定位到真实人物的信息

**禁止发布的内容**：
- 侮辱、报复、控诉式内容
- 把他人真实身份作为卖点
- 引导网暴或猎奇的内容

### 3.2 可以写什么

- 礼物本身的描述（颜色、材质、品牌、使用感受）
- 关系类型（恋人、夫妻、朋友、家人等，宽泛描述）
- 自己的感受（遗憾、感谢、释怀、放下等）
- 送礼时的场景（节假日、旅行、共同记忆等，不暴露具体地点）
- 对礼物下一站的期待

### 3.3 表单提示文案

在发布表单旁边，有一段温柔提示：

> *不必写得戏剧化。只要诚实地说出：它曾经来过，现在该走了。*

## 4. 常见问题

**Q：发布后为什么看不到我的礼物？**
A：礼物提交后进入 `pending_review` 状态，需要 Admin 审核后才会展示。请等待 Admin 处理，或联系 Admin token 持有者加速审核。

**Q：收藏的礼物不见了？**
A：收藏数据保存在 localStorage，清除浏览器数据会导致丢失。请确保浏览器未开启无痕模式自动清理。

**Q：草稿丢失了怎么办？**
A：草稿保存在 localStorage，清除浏览器数据会导致草稿丢失。当前不支持服务器端草稿同步。

**Q：如何删除已发布的礼物？**
A：在"我的发布"页面点击归档（Archive）按钮，礼物会从公开流中移除但不会物理删除，可在归档区恢复。

**Q：匿名身份可以恢复吗？**
A：匿名身份存储在 localStorage 中的 JWT，关闭浏览器后再次打开需要重新创建匿名身份，原有的发布和收藏记录会丢失。建议在内测阶段不要频繁清除 localStorage。

## 5. 免责声明

Aftergift 是一个**本地原型**，不含真实支付、物流、交易撮合功能。用户在实际使用中：
- 发布真实联系方式可能导致隐私泄露
- 线下交易存在安全风险
- AI 审核不能替代人工判断

**请勿将 Aftergift 作为唯一的事实记录工具**，所有数据存储在本地 SQLite 和 localStorage 中，服务器不负责持久化。
