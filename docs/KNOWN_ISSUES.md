# Aftergift 已知问题与限制

> 当前版本：Phase 2H-2 Local Beta
> 最后更新：2026-05-16

## 架构限制

1. **后端仅本地运行**
   - 无公网部署，内测需在同一网络或本地访问
   - 无 HTTPS，token 传输存在中间人风险

2. **SQLite 不适合高并发**
   - 文件级锁，并发写操作可能冲突
   - 单文件存储，不适合水平扩展
   - 评估迁移 PostgreSQL 条件：并发用户 > 50 或 P95 延迟 > 500ms

3. **localStorage token 有 XSS 风险**
   - JWT token 存储在浏览器 localStorage
   - XSS 攻击可窃取 token
   - 缓解：无用户输入渲染为 HTML（纯文本展示）

4. **无 refresh token**
   - JWT 过期后需重新创建匿名身份
   - 当前 TTL 7 天，内测期间足够

## 功能缺失

5. **无真实支付和物流**
   - 仅表达交易意向，无担保交易
   - 用户需自行协商线下交接

6. **无私信功能**
   - 无法联系发布者
   - 内测期间通过其他渠道（如 Telegram）沟通

7. **OpenAI provider 默认关闭**
   - 审核使用 mock provider（规则引擎）
   - 真实 AI 审核需手动配置 OPENAI_API_KEY

8. **搜索不是全文检索**
   - 基于 LIKE %keyword% 的简单匹配
   - 无分词、无相关性排序
   - 大数据量时性能下降

9. **推荐算法尚未实现**
   - 无「猜你喜欢」或「相似故事」
   - 内容发现依赖搜索和筛选

10. **草稿自动保存只在本设备有效**
    - 基于 localStorage，换浏览器/清缓存/换设备丢失
    - 无后端 draft 表持久化

## 前端限制

11. **Static 模式功能受限**
    - 无搜索、无收藏、无我的发布
    - 仅展示静态 JSON 数据
    - 完整功能需 `?api=local`

12. **移动端适配待优化**
    - 基本可用，但部分 Modal 在窄屏下体验一般
    - 长表单滚动体验待改进

## 安全与隐私

13. **审核层非人工**
    - mock provider 基于正则规则，可能误判
    - 高风险内容依赖 admin 人工复核

14. **无 IP 限流**
    - 内测期间无防刷机制
    - 恶意用户可批量创建内容

15. **无数据加密**
    - 数据库文件未加密
    - 备份文件需妥善保管

## 计划修复时间线

| 问题 | 预计修复阶段 |
|------|-------------|
| 无 refresh token | Phase 2I |
| 搜索性能 | Phase 2I 或 PostgreSQL 迁移 |
| 推荐算法 | Phase 2I |
| 私信功能 | Phase 3A |
| 支付/物流 | Phase 3B |
| PostgreSQL 迁移 | Phase 2G-4（满足条件时） |
